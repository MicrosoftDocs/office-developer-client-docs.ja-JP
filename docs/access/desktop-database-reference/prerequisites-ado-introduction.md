ADO はさまざまなソースのデータ アクセスに使用されるため、読者には、リレーショナル データベース管理システムの基本的な概念、オンライン分析処理 (OLAP) の概念、およびインターネットやインターネット プロトコルの基本的な知識も必要になります。

ADO は、Microsoft データ アクセス (UDA) 戦略の一部です。 (UDA の詳細については、この章の後半の[データを Access で ADO の役割](the-role-of-ado-in-microsoft-data-access.md)を参照してください)。したがって、OLE DB テクノロジと相互運用します。 OLE DB は、Microsoft Component Object Model (COM) に基づいています。 したがって、COM を理解していると、ADO のさらに高度な概念を理解するのに役立ちます。

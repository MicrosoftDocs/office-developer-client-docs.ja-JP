<span data-ttu-id="b8a3f-105">レコード キーと検索キーは、多くの MAPI オブジェクトに割り当てられているバイナリ識別子です。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-105">Record keys and search keys are binary identifiers that are assigned to many MAPI objects.</span></span> <span data-ttu-id="b8a3f-106">オブジェクトのエントリ識別子とは異なり、レコードまたは検索キーは直接同等で、送信可能です。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-106">Unlike an object's entry identifier, its record or search key is directly comparable as well as transmittable.</span></span> 
  
## <a name="record-keys"></a><span data-ttu-id="b8a3f-107">レコード キー</span><span class="sxs-lookup"><span data-stu-id="b8a3f-107">Record Keys</span></span>

<span data-ttu-id="b8a3f-108">レコード キーを使用して、2 つのオブジェクトを比較します。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-108">A record key is used to compare two objects.</span></span> <span data-ttu-id="b8a3f-109">メッセージ ストア オブジェクトとアドレス帳オブジェクトには、レコード キーが必要です。このキーは、PR_RECORD_KEY **(** [PidTagRecordKey](pidtagrecordkey-canonical-property.md)) プロパティに格納されます。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-109">Message store and address book objects must have record keys, which are stored in their **PR_RECORD_KEY** ([PidTagRecordKey](pidtagrecordkey-canonical-property.md)) property.</span></span> <span data-ttu-id="b8a3f-110">レコード キーはオブジェクトを識別し、データを識別しないので、オブジェクトのすべてのインスタンスには一意のレコード キーがあります。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-110">Because a record key identifies an object and not its data, every instance of an object has a unique record key.</span></span> <span data-ttu-id="b8a3f-111">フォルダーとメッセージのレコード キーのスコープは、メッセージ ストアです。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-111">The scope of a record key for folders and messages is the message store.</span></span> <span data-ttu-id="b8a3f-112">アドレス帳コンテナー、メッセージング ユーザー、配布リストのスコープは、統合アドレス帳で使用するために MAPI によって提供される一連のトップ レベル のコンテナーです。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-112">The scope for address book containers, messaging users, and distribution lists is the set of top-level containers provided by MAPI for use in the integrated address book.</span></span>
  
<span data-ttu-id="b8a3f-113">レコード キーは、別のリソースで複製できます。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-113">Record keys can be duplicated in another resource.</span></span> <span data-ttu-id="b8a3f-114">たとえば、2 つの異なるメッセージ ストア内の異なるメッセージに同じレコード キーを設定できます。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-114">For example, different messages in two different message stores can have the same record key.</span></span> <span data-ttu-id="b8a3f-115">これは、長期的なエントリ識別子とは異なります。長期エントリ識別子にはサービス プロバイダーへの参照が含まれているため、スコープが広くなります。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-115">This is different from long-term entry identifiers; because long-term entry identifiers contain a reference to the service provider, they have a wider scope.</span></span> <span data-ttu-id="b8a3f-116">メッセージ ストアのレコード キーは、スコープ内で長期的なエントリ識別子と似ています。すべてのメッセージ ストア プロバイダーで一意である必要があります。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-116">A message store's record key is similar in scope to a long-term entry identifier; it should be unique across all message store providers.</span></span> <span data-ttu-id="b8a3f-117">この一意性を確保するために、メッセージ ストア プロバイダーは通常、レコード キーを **PR_MDB_PROVIDER** ([PidTagStoreProvider](pidtagstoreprovider-canonical-property.md)) プロパティとメッセージ ストアに固有の識別子の組み合わせの値に設定します。</span><span class="sxs-lookup"><span data-stu-id="b8a3f-117">To ensure this uniqueness, message store providers typically set their record key to a value that is the combination of their **PR_MDB_PROVIDER** ([PidTagStoreProvider](pidtagstoreprovider-canonical-property.md)) property and an identifier that is unique to the message store.</span></span>
